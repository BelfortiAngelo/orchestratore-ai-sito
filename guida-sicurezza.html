Cosa puo fare Claude sul tuo computer

Quando usi Claude Code, non stai solo chattando. Claude ha accesso al terminale del tuo computer e puo eseguire comandi reali: creare file, modificarli, cancellarli, mandare codice su GitHub, installare software.

Nella maggior parte dei casi e esattamente quello che vuoi. Ma ci sono comandi che non dovresti mai eseguire per sbaglio — e Claude, se non glielo impedisci, potrebbe eseguirli senza cattive intenzioni ma con conseguenze difficili da invertire.

I comandi piu pericolosi

git reset --hard — cancella tutte le modifiche non salvate, senza possibilita di recupero.
git push --force — sovrascrive la storia su GitHub, puo cancellare il lavoro dei tuoi collaboratori.
rm -rf — elimina cartelle intere in modo permanente.
Accesso a file .env — questi file contengono password e chiavi API. Non devono mai finire in un commit.

Hook vs Sandbox: due approcci diversi

Esistono due modi per proteggere il terminale quando lavori con Claude Code:

  • Gli hook sono come un semaforo: controllano ogni comando prima che venga eseguito e lo bloccano se corrisponde a una lista nera. Facili da installare, funzionano su Windows.
  • Il sandbox e come un muro fisico: crea un recinto a livello di sistema operativo dove Claude non puo uscire dalla cartella di lavoro ne fare richieste di rete non autorizzate. Molto piu robusto degli hook, ma al momento non supportato su Windows nativo (solo macOS e Linux).

Su Windows, gli hook sono la soluzione migliore disponibile adesso. Quando arrivera il supporto sandbox per Windows, sara il primo upgrade da fare.

Come installare gli hook su Windows

Servono due file PowerShell e una modifica al settings.json di Claude Code. Niente da installare, niente da scaricare — solo file di testo.

Passo 1 — Crea la cartella hooks

C:\Users\[tuonome]\.claude\hooks\

Se la cartella .claude non esiste, creala. Dentro, crea la sottocartella hooks.

Passo 2 — Crea il file check-bash.ps1

Questo script blocca i comandi bash pericolosi:

# check-bash.ps1 — blocca comandi pericolosi
$input_json = $input | ConvertFrom-Json
$command = $input_json.tool_input.command

$blocked = @(
  "push --force",
  "push -f",
  "reset --hard",
  "rm -rf"
)

foreach ($pattern in $blocked) {
  if ($command -match [regex]::Escape($pattern)) {
    @{ decision = "block"; reason = "Comando bloccato dall hook di sicurezza: $pattern" } | ConvertTo-Json
    exit 0
  }
}

@{ decision = "approve" } | ConvertTo-Json

Passo 3 — Crea il file check-files.ps1

Questo script protegge i file .env:

# check-files.ps1 — protegge file sensibili
$input_json = $input | ConvertFrom-Json
$path = $input_json.tool_input.path

if ($path -match "\\.env") {
  @{ decision = "block"; reason = "Accesso a file .env bloccato" } | ConvertTo-Json
  exit 0
}

@{ decision = "approve" } | ConvertTo-Json

Passo 4 — Registra gli hook nel settings.json

Apri (o crea) il file ~/.claude/settings.json e aggiungi questa sezione:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -File ~/.claude/hooks/check-bash.ps1"
          }
        ]
      },
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -File ~/.claude/hooks/check-files.ps1"
          }
        ]
      }
    ]
  }
}

Riavvia Claude Code. Da questo momento, ogni volta che Claude prova a eseguire uno dei comandi bloccati, lo script interviene prima che venga eseguito.

I 4 layer di sicurezza

Gli hook sono solo il secondo layer. Ecco come si struttura la sicurezza completa, dal piu semplice al piu robusto:

  • Layer 1 — CLAUDE.md: scrivi esplicitamente cosa Claude non deve fare. Il modello puo dimenticare, ma e un punto di partenza. Stato: da aggiungere ai progetti critici.
  • Layer 2 — Hook di sicurezza: blocca comandi specifici a livello di esecuzione. Stato: installati.
  • Layer 3 — Branch workflow: Claude lavora sempre su un branch separato, mai direttamente su main. Sblocca con GitHub MCP. Stato: da configurare.
  • Layer 4 — Sandbox OS-level: recinto filesystem a livello di sistema operativo. Stato: in attesa di supporto Windows nativo.
La lezione

Non aspettare di capire tutto prima di mettere le protezioni. Gli hook si installano in 15 minuti e proteggono dai danni piu comuni. Fallo prima di dare a Claude l'accesso a qualsiasi progetto che ti importa.
